## 2. 功能模块说明

### 2.1 知识问答系统 (QA)
- **路径**: `/api/v1/qa`
- **功能**: 问答、对话历史管理
- **核心接口**:
  - `POST /ask` - 提问并获取回答
  - `GET /history` - 获取对话历史

### 2.2 感知系统 (Perception)
- **路径**: `/api/v1/perception`
- **功能**: 学习路径感知、认知模型
- **核心接口**:
  - `POST /perceive` - 记录学习事件
  - `GET /state` - 获取当前状态

### 2.3 探索系统 (Exploration)
- **路径**: `/api/v1/exploration`
- **功能**: 知识缺口检测、自动探索
- **核心接口**:
  - `GET /gaps` - 检测知识缺口
  - `POST /run` - 运行探索循环

### 2.4 规划系统 (Planning)
- **路径**: `/api/v1/planning`
- **功能**: 学习计划生成、任务管理
- **核心接口**:
  - `POST /plans` - 创建学习计划
  - `GET /plans` - 列出计划

### 2.5 知识卡片系统 (Knowledge Cards) ⭐新增
- **路径**: `/api/v1/knowledge`
- **功能**: 学习目标、知识卡片、测试、复习
- **核心接口**:
  - `POST /goals` - 创建学习目标
  - `POST /goals/{id}/populate` - 自动填充知识卡片
  - `GET /goals/{id}/progress` - 查看学习进度
  - `POST /goals/{id}/quiz` - 生成测试题
  - `POST /mastery/update` - 更新掌握度
  - `GET /goals/{id}/due-review` - 获取到期复习

## 3. 完整功能测试流程

### 第一步：启动系统
```bash
debug_app_start.bat
```
等待看到 "✅ 系统启动完成！"

### 第二步：打开API文档
浏览器访问: http://127.0.0.1:8000/docs

### 第三步：测试健康检查
1. 在文档中找到 `GET /health`
2. 点击 "Try it out"
3. 点击 "Execute"
4. 检查返回结果:
   ```json
   {
     "status": "ok",
     "app": "自主智能学习系统",
     "background_tasks": true
   }
   ```

### 第四步：测试知识卡片系统（推荐）

#### 4.1 创建学习目标
- 接口: `POST /api/v1/knowledge/goals`
- 请求体:
  ```json
  {
    "description": "学习10个常用汉字"
  }
  ```
- 预期返回:
  ```json
  {
    "id": 1,
    "description": "学习10个常用汉字",
    "goal_type": "characters",
    "unit_name": "汉字",
    "estimated_count": 10,
    "total_units": 0,
    "populated_count": 0,
    "populated": false
  }
  ```
- **说明**: 系统会自动识别类型为"characters"

#### 4.2 填充知识卡片
- 接口: `POST /api/v1/knowledge/goals/{goal_id}/populate`
- 将 `{goal_id}` 替换为上面返回的 `id` (如: 1)
- 点击 "Execute"
- **等待**：这可能需要1-2分钟（调用LLM生成内容）
- 预期返回:
  ```json
  {
    "success": true,
    "total_units": 10,
    "generated": 10,
    "failed": 0,
    "goal_type": "characters"
  }
  ```

#### 4.3 查看学习进度
- 接口: `GET /api/v1/knowledge/goals/{goal_id}/progress`
- 预期返回:
  ```json
  {
    "total_units": 10,
    "learned": 0,
    "unseen": 10,
    "learning": 0,
    "familiar": 0,
    "mastered": 0,
    "completion_rate": 0,
    "mastery_rate": 0,
    "avg_score": 0
  }
  ```

#### 4.4 生成测试题
- 接口: `POST /api/v1/knowledge/goals/{goal_id}/quiz`
- 参数:
  - `count`: 3 (题目数量)
  - `mode`: "unseen" (测试未学习的内容)
- 预期返回:
  ```json
  {
    "quizzes": [
      {
        "card_id": 1,
        "question": "「一」怎么读？",
        "answer": "yī",
        "hint": "最简单的汉字",
        "type": "reading"
      },
      ...
    ]
  }
  ```

#### 4.5 更新掌握度
- 接口: `POST /api/v1/knowledge/mastery/update`
- 请求体:
  ```json
  {
    "card_id": 1,
    "score": 0.8,
    "response_time": 5.0
  }
  ```
- **说明**:
  - `card_id`: 从测试题结果中获取
  - `score`: 0.0-1.0 (0.8表示80%正确)
  - `response_time`: 答题用时（秒）
- 预期返回:
  ```json
  {
    "status": "familiar",
    "score": 1.0,
    "interval_days": 1,
    "next_review": "2026-03-03T13:40:00"
  }
  ```

#### 4.6 再次查看进度
- 接口: `GET /api/v1/knowledge/goals/{goal_id}/progress`
- 应该能看到 `learned` 增加，`completion_rate` > 0

### 第五步：测试问答系统

#### 5.1 提问

## 4. API接口使用指南

### 4.1 使用cURL测试

#### 创建目标并填充
```bash
# 1. 创建目标
curl -X POST "http://127.0.0.1:8000/api/v1/knowledge/goals" \
  -H "Content-Type: application/json" \
  -d '{"description": "学习5个常用汉字"}'

# 2. 填充知识卡片 (假设goal_id=1)
curl -X POST "http://127.0.0.1:8000/api/v1/knowledge/goals/1/populate"

# 3. 查看进度
curl "http://127.0.0.1:8000/api/v1/knowledge/goals/1/progress"

# 4. 生成测试题
curl -X POST "http://127.0.0.1:8000/api/v1/knowledge/goals/1/quiz?count=3&mode=unseen"

# 5. 更新掌握度
curl -X POST "http://127.0.0.1:8000/api/v1/knowledge/mastery/update" \
  -H "Content-Type: application/json" \
  -d '{"card_id": 1, "score": 0.8, "response_time": 5.0}'
```

### 4.2 使用Python脚本测试
```python
import requests
import json

BASE_URL = "http://127.0.0.1:8000/api/v1"

# 1. 创建目标
response = requests.post(f"{BASE_URL}/knowledge/goals",
    json={"description": "学习5个汉字"})
goal_id = response.json()["id"]
print(f"创建目标 ID: {goal_id}")

# 2. 填充知识卡片
response = requests.post(f"{BASE_URL}/knowledge/goals/{goal_id}/populate")
print(f"填充结果: {response.json()}")

# 3. 查看进度
response = requests.get(f"{BASE_URL}/knowledge/goals/{goal_id}/progress")
print(f"进度: {response.json()}")

# 4. 生成测试题
response = requests.post(
    f"{BASE_URL}/knowledge/goals/{goal_id}/quiz",
    params={"count": 2, "mode": "unseen"}
)
quizzes = response.json()["quizzes"]
print(f"测试题: {quizzes}")

# 5. 更新掌握度
for quiz in quizzes:
    response = requests.post(f"{BASE_URL}/knowledge/mastery/update",
        json={
            "card_id": quiz["card_id"],
            "score": 0.8,
            "response_time": 5.0
        })
    print(f"更新掌握度: {response.json()}")
```

## 5. 常见问题

### Q1: API调用返回500错误？
**A**: 检查服务器日志，常见原因：
- LLM API密钥未配置或无效
- 数据库文件权限问题
- 向量存储服务未启动

### Q2: 知识卡片填充很慢？
**A**: 这是正常的，因为需要：
1. 调用LLM生成知识单元列表
2. 逐个调用LLM生成知识卡片内容
3. 每个API调用有延迟限制

建议：
- 减少目标数量（如从3500改为50）
- 使用更快的LLM模型

### Q3: 如何重置系统？
**A**: 删除数据库文件：
```bash
del als.db
```
然后重新启动，系统会自动创建新数据库。

### Q4: SM-2算法如何工作？
**A**: 系统使用SuperMemo 2算法：
- 答题正确率≥60%：增加复习间隔
- 答题正确率<60%：重置间隔为1天
- 考虑答题速度：>30秒会降低评分

复习间隔计算：
```
第1次复习：1天后
第2次复习：6天后
第3次及以后：interval × ease_factor
最大间隔：365天
```

### Q5: 如何修改豆包API配置？
**A**: 编辑 `.env` 文件：
```
OPENAI_API_KEY=你的密钥
OPENAI_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
LLM_MODEL=doubao-seed-1-8-251228
```

### Q6: 后台任务在做什么？
**A**:
- **向量同步**: 每5分钟同步知识到向量数据库
- **探索任务**: 每1小时检测知识缺口
